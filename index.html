Webflow Blog CMS API Test

Use this page to test pulling blog posts from Webflow CMS, then give the Framer dev the server-side fetch pattern below.

What Framer Needs

  1. Webflow API token with CMS:read scope.
  2. Blog Collection ID from Webflow.
  3. Endpoint: GET https://api.webflow.com/v2/collections/:collection_id/items
  4. Query: limit=100, offset=0, sortBy=lastPublished, sortOrder=desc.

Important

Do not expose the Webflow token in production client-side code. Put it in a Framer/serverless backend route or secure integration layer, then return safe blog JSON to the page.

Server-Side Fetch

const WEBFLOW_TOKEN = process.env.WEBFLOW_TOKEN
const BLOG_COLLECTION_ID = process.env.WEBFLOW_BLOG_COLLECTION_ID

export async function getWebflowBlogPosts() {
  const params = new URLSearchParams({
    limit: "100",
    offset: "0",
    sortBy: "lastPublished",
    sortOrder: "desc",
  })

  const response = await fetch(
    `https://api.webflow.com/v2/collections/${BLOG_COLLECTION_ID}/items?${params}`,
    {
      headers: {
        Authorization: `Bearer ${WEBFLOW_TOKEN}`,
        accept: "application/json",
      },
    }
  )

  if (!response.ok) {
    throw new Error(`Webflow CMS error: ${response.status}`)
  }

  const data = await response.json()

  return data.items.map((item) => ({
    id: item.id,
    name: item.fieldData.name,
    slug: item.fieldData.slug,
    url: `/blog/${item.fieldData.slug}`,
    publishedAt: item.lastPublished,
    updatedAt: item.lastUpdated,
    fields: item.fieldData,
  }))
}

curl -G "https://api.webflow.com/v2/collections/BLOG_COLLECTION_ID/items" \
  -H "Authorization: Bearer WEBFLOW_TOKEN" \
  -H "accept: application/json" \
  -d limit=100 \
  -d offset=0 \
  -d sortBy=lastPublished \
  -d sortOrder=desc
